Value Type Description Registrant Reference
result set truncated due to authorization notice and remark type The list of results does not contain all results due to lack of authorization. This may indicate to some clients that proper authorization will yield a longer result set. [IESG] [RFC-ietf-weirds-json-response-14]
result set truncated due to excessive load notice and remark type The list of results does not contain all results due to excessively heavy load on the server. This may indicate to some clients that requerying at a later time will yield a longer result set. [IESG] [RFC-ietf-weirds-json-response-14]
result set truncated due to unexplainable reasons notice and remark type The list of results does not contain all results for an unexplainable reason. This may indicate to some clients that requerying for any reason will not yield a longer result set. [IESG] [RFC-ietf-weirds-json-response-14]
object truncated due to authorization notice and remark type The object does not contain all data due to lack of authorization. [IESG] [RFC-ietf-weirds-json-response-14]
object truncated due to excessive load notice and remark type The object does not contain all data due to excessively heavy load on the server. This may indicate to some clients that requerying at a later time will yield all data of the object. [IESG] [RFC-ietf-weirds-json-response-14]
object truncated due to unexplainable reasons notice and remark type The object does not contain all data for an unexplainable reason. [IESG] [RFC-ietf-weirds-json-response-14]
validated status Signifies that the data of the object instance has been found to be accurate. This type of status is usually found on entity object instances to note the validity of identifying contact information. [IESG] [RFC-ietf-weirds-json-response-14]
renew prohibited status Renewal or reregistration of the object instance is forbidden. [IESG] [RFC-ietf-weirds-json-response-14]
update prohibited status Updates to the object instance are forbidden. [IESG] [RFC-ietf-weirds-json-response-14]
transfer prohibited status Transfers of the registration from one registrar to another are forbidden. This type of status normally applies to DNR domain names. [IESG] [RFC-ietf-weirds-json-response-14]
delete prohibited status Deletion of the registration of the object instance is forbidden. This type of status normally applies to DNR domain names. [IESG] [RFC-ietf-weirds-json-response-14]
proxy status The registration of the object instance has been performed by a third party. This is most commonly applied to entities. [IESG] [RFC-ietf-weirds-json-response-14]
private status The information of the object instance is not designated for public consumption. This is most commonly applied to entities. [IESG] [RFC-ietf-weirds-json-response-14]
removed status Some of the information of the object instance has not been made available and has been removed. This is most commonly applied to entities. [IESG] [RFC-ietf-weirds-json-response-14]
obscured status Some of the information of the object instance has been altered for the purposes of not readily revealing the actual information of the object instance. This is most commonly applied to entities. [IESG] [RFC-ietf-weirds-json-response-14]
associated status The object instance is associated with other object instances in the registry. This is most commonly used to signify that a nameserver is associated with a domain or that an entity is associated with a network resource or domain. [IESG] [RFC-ietf-weirds-json-response-14]
active status The object instance is in use. For domain names, it signifies that the domain name is published in DNS. For network and autnum registrations it signifies that they are allocated or assigned for use in operational networks. This maps to the Extensible Provisioning Protocol (EPP) [RFC5730] 'OK' status. [IESG] [RFC-ietf-weirds-json-response-14]
inactive status The object instance is not in use. See 'active'. [IESG] [RFC-ietf-weirds-json-response-14]
locked status Changes to the object instance cannot be made, including the association of other object instances. [IESG] [RFC-ietf-weirds-json-response-14]
pending create status A request has been received for the creation of the object instance but this action is not yet complete. [IESG] [RFC-ietf-weirds-json-response-14]
pending renew status A request has been received for the renewal of the object instance but this action is not yet complete. [IESG] [RFC-ietf-weirds-json-response-14]
pending transfer status A request has been received for the transfer of the object instance but this action is not yet complete. [IESG] [RFC-ietf-weirds-json-response-14]
pending update status A request has been received for the update or modification of the object instance but this action is not yet complete. [IESG] [RFC-ietf-weirds-json-response-14]
pending delete status A request has been received for the deletion or removal of the object instance but this action is not yet complete. For domains, this might mean that the name is no longer published in DNS but has not yet been purged from the registry database. [IESG] [RFC-ietf-weirds-json-response-14]
registration event action The object instance was initially registered. [IESG] [RFC-ietf-weirds-json-response-14]
reregistration event action The object instance was registered subsequently to initial registration. [IESG] [RFC-ietf-weirds-json-response-14]
last changed event action An action noting when the information in the object instance was last changed. [IESG] [RFC-ietf-weirds-json-response-14]
expiration event action The object instance has been removed or will be removed at a pre-determined date and time from the registry. [IESG] [RFC-ietf-weirds-json-response-14]
deletion event action The object instance was removed from the registry at a point in time that was not pre-determined. [IESG] [RFC-ietf-weirds-json-response-14]
reinstantiation event action The object instance was reregistered after having been removed from the registry. [IESG] [RFC-ietf-weirds-json-response-14]
transfer event action The object instance was transferred from one registrant to another. [IESG] [RFC-ietf-weirds-json-response-14]
locked event action The object instance was locked (see the 'locked' status). [IESG] [RFC-ietf-weirds-json-response-14]
unlocked event action The object instance was unlocked (see the 'locked' status). [IESG] [RFC-ietf-weirds-json-response-14]
registrant role The entity object instance is the registrant of the registration. In some registries, this is known as a maintainer. [IESG] [RFC-ietf-weirds-json-response-14]
technical role The entity object instance is a technical contact for the registration. [IESG] [RFC-ietf-weirds-json-response-14]
adminstrative role The entity object instance is an administrative contact for the registration. [IESG] [RFC-ietf-weirds-json-response-14]
abuse role The entity object instance handles network abuse issues on behalf of the registrant of the registration. [IESG] [RFC-ietf-weirds-json-response-14]
billing role The entity object instance handles payment and billing issues on behalf of the registrant of the registration. [IESG] [RFC-ietf-weirds-json-response-14]
registrar role The entity object instance represents the authority responsible for the registration in the registry. [IESG] [RFC-ietf-weirds-json-response-14]
reseller role The entity object instance represents a third party through which the registration was conducted (i.e. not the registry or registrar). [IESG] [RFC-ietf-weirds-json-response-14]
sponsor role The entity object instance represents a domain policy sponsor, such as an ICANN approved sponsor. [IESG] [RFC-ietf-weirds-json-response-14]
proxy role The entity object instance represents a proxy for another entity object, such as a registrant. [IESG] [RFC-ietf-weirds-json-response-14]
notifications role An entity object instance designated to receive notifications about association object instances. [IESG] [RFC-ietf-weirds-json-response-14]
noc role The entity object instance handles communications related to a network operations center (NOC). [IESG] [RFC-ietf-weirds-json-response-14]
registered domain variant relation The variant names are registered in the registry. [IESG] [RFC-ietf-weirds-json-response-14]
unregistered domain variant relation The variant names are not found in the registry. [IESG] [RFC-ietf-weirds-json-response-14]
registration restricted domain variant relation Registration of the variant names is restricted to certain parties or within certain rules. [IESG] [RFC-ietf-weirds-json-response-14]
open registration domain variant relation Registration of the variant names is available to generally qualified registrants. [IESG] [RFC-ietf-weirds-json-response-14]
conjoined domain variant relation Registration of the variant names occurs automatically with the registration of the containing domain registration. [IESG] [RFC-ietf-weirds-json-response-14]
